# Воксельна система Voxeterra

## Реалізовано

### 🎮 Воксельна система
- **Chunk система**: 16×16×16 вокселів на чанк
- **Типи вокселів**: AIR, STONE, DIRT, GRASS, SAND, WATER
- **World manager**: Hash map для управління чанками

### 🔧 Жадібний бінарний мешінг (Greedy Binary Meshing)
Алгоритм оптимізації геометрії:
- Об'єднує сусідні однакові вокселі в великі quad'и
- Розширення по 2 осях (width × height)
- Генерація тільки видимих граней
- Обробка всіх 6 напрямків

**Результат**: Замість 36 трикутників на воксель → 2-12 трикутників на групу вокселів

### 🌍 Генерація світу
- **Розмір**: 5×1×5 чанків (80×16×80 вокселів)
- **Структура**:
  - Глибокі шари: камінь
  - Середні шари: земля
  - Поверхня: плями різних типів (процедурний шум)

### 🎨 Рендеринг
- **Золоте правило**: один воксель = один колір
- **Кольори**:
  - 🪨 Камінь (STONE): сірий `#808080`
  - 🟫 Земля (DIRT): коричневий `#8C5A32`
  - 🟩 Трава (GRASS): зелений `#4DB84D`
  - 🟨 Пісок (SAND): жовтий `#E6D999`
  - 🟦 Вода (WATER): синій `#3366CC`
- **Lighting**: простий directional light з ambient

### 📊 Статистика
- 25 чанків згенеровано
- ~700-1000 вершин на чанк
- ~1000-1500 індексів на чанк
- Загалом: ~22,000 вершин, ~33,000 індексів

## Архітектура

```
World
  └── Chunks (hash map)
       └── Voxels [16×16×16]
            └── VoxelType (uint8_t)

Chunk::generateMesh()
  └── Greedy Binary Meshing
       └── VoxelVertex[] + uint32_t[]
            └── Buffer (GPU)
```

## Технічні деталі

### VoxelVertex структура
```cpp
struct VoxelVertex {
    glm::vec3 position;   // 12 bytes
    glm::vec3 normal;     // 12 bytes
    glm::vec2 texCoord;   // 8 bytes
    uint32_t voxelType;   // 4 bytes
};  // Total: 36 bytes
```

### Жадібний алгоритм
1. Для кожної грані (6 напрямків)
2. Для кожного воксела
3. Якщо воксель solid і сусід - повітря:
   - Розширюємо по осі X (width)
   - Розширюємо по осі Y (height)
   - Створюємо quad (2 трикутники)
   - Позначаємо вокселі як оброблені

### Оптимізації
- Тільки видимі грані
- Об'єднання однакових вокселів
- Host-visible buffers (для швидкого оновлення)
- Один draw call на чанк
